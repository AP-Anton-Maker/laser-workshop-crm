# Anton Maker CRM

Полнофункциональная система управления производством для лазерной мастерской Anton Maker (anton-maker.ru).

## 🚀 Особенности

- **Интеграция с ВКонтакте**: Приём заказов через вебхук, автоматическое взаимодействие с клиентами
- **Интерактивный калькулятор**: Онлайн-расчёт стоимости с возможностью настройки в админке
- **Управление производством**: Отслеживание статусов заказов, маршрутные листы в PDF
- **Управление материалами**: Гибкая настройка материалов и дополнительных услуг
- **Мониторинг KPI**: Статистика по производительности и отходам
- **Оптимизированная БД**: SQLite с WAL-режимом для NVMe SSD

## 🛠️ Технологии

- **Backend**: Python 3.10+, Django 4.2 LTS
- **Frontend**: HTML/CSS (Tailwind CSS), JavaScript
- **БД**: SQLite3 с оптимизациями WAL
- **Сервер**: Gunicorn + Nginx
- **Админка**: django-unfold
- **ВКонтакте**: webhook API
- **PDF**: WeasyPrint

## 📋 Модели данных

- **Client**: Информация о клиентах (имя, телефон, VK ID, состояние бота)
- **Material**: Справочник материалов с типами обработки и ценами
- **ServiceOption**: Дополнительные услуги (чекбоксы в калькуляторе)
- **Order**: Заказы клиентов (статусы, файлы, расчёты)
- **ProductionKPI**: Метрики производительности

## 🎨 Функционал

### Калькулятор
- Интерактивный расчёт стоимости на основе материалов и размеров
- Поддержка различных типов обработки (гравировка, резка, время, глубокая)
- Дополнительные услуги (чекбоксы)
- Административный калькулятор в панели управления

### Бот ВКонтакте
- Приём заказов через вебхук
- Машина состояний (START → WAIT_DESC → WAIT_FILE → COMPLETED)
- Автоматическая генерация маршрутов
- Уведомления администратору

### Админ-панель
- Управление материалами и услугами
- Просмотр и редактирование заказов
- Печать маршрутных листов в PDF
- Дашборд KPI

## 🚀 Установка и запуск

### Требования
- Debian 12 (Proxmox LXC)
- Python 3.10+
- Система без Docker (прямая установка)

### Быстрый запуск
```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/anton-maker.git
cd anton-maker

# 2. Настройте переменные окружения
cp .env.example .env
nano .env  # Укажите VK_TOKEN, SECRET_KEY, ADMIN_VK_ID и другие

# 3. Запустите автоматическую установку
bash deploy/install.sh

# 4. Следуйте инструкциям по настройке домена и SSL
```

Подробная инструкция по развёртыванию в файле `DEPLOY.md`.

## 🔧 Переменные окружения

- `SECRET_KEY`: Секретный ключ Django
- `VK_APP_ID`: ID приложения ВКонтакте
- `VK_TOKEN`: Сервисный токен сообщества
- `VK_CONFIRMATION_CODE`: Код подтверждения вебхука
- `ADMIN_VK_ID`: ID администратора ВКонтакте
- `DEBUG`: Режим отладки (True/False)
- `ALLOWED_HOSTS`: Разрешённые хосты (через запятую)

## 📊 KPI и отчёты

- **OEE (Overall Equipment Effectiveness)**: Общая эффективность оборудования
- **Метрики отходов**: Уровень отходов по материалам
- **Ежедневные отчёты**: Автоматическая отправка в ВК
- **Аналитика нагрузки**: Графики загрузки станков
